Dependencies:

  • ui.core.js
  • ui.draggable.js
  • ui.resizable.js

JS call:


     $(".container").containerize();

HTML constructor:


<div id="cont1" class="container" style=" top:130px; left: 300px; height: auto" data-icon="css/icons/browser.png" data-drag= data-resize=true  data-dock="dock" data-collapse=true data-close=true data-containment="document" data-modal=true data-buttons="fullscreen,close">
        <h2>[title]</h2>
        <p>[content]</p>
    </div>

Parameters:

Those are the general attribute for all the containers you are initializing with a single call:

  • containment:“document”, (string) the container containment once dragged; can be any jQuery DOM Element
  • mantainOnWindow:true (boolean) define if the container should stay inside the screen view
  • effectDuration:300 (int) time in milliseconds for the effect
  • zIndexContext:‘auto’ (string) ‘auto’ means the container will have the highest z-index once active; you can pass also a CLASS attribute to set wich element should be affected by the bringToFfront function

Callbacks:

  • onLoad:function(o){}, (function) fired once the container is initialized
  • onCollapse:function(o){}, (function) fired on container collapse event
  • onBeforeIconize:function(o){}, (function) fired just before the container iconize event
  • onIconize:function(o){}, (function) fired on container iconize event
  • onClose: function(o){}, (function) fired on container close event
  • onBeforeClose: function(o){}, (function) fired before the container close event
  • onResize: function(o,w,h){}, (function) fired on container resize event
  • onDrag: function(o,x,y){}, (function) fired on container drag event
  • onRestore:function(o){}, (function) fired on container restore event
  • onFullScreen:function(o){}, (function) fired on container fullscreen event
o: the container object – w: the container width – h: the container height – x: the container left position – y: the container top position

Configure each container:

You can configure each container by adding custom attributes to the HTML constructor. This will apply the corresponding defined method to the container.

Built in custom attributes:

  • data-icon: (string) the path to the icon.
  • data-skin: (string) the name of the skin for this container (available: black); if not set it get the default white skin.
  • data-drag: (boolean) define if the container is draggable or not.
  • data-resize: (boolean) define if the container is resizable or not.
  • data-dock: (string) the ID of the dock for this container; if no dock ID has been specified the container will iconize on the left side of the window.
  • data-collapsible: (boolean) define if the container is collapsable or not.
  • data-collapse: (boolean) define if the container is collapsed or not on initialization.
  • data-close: (boolean) define if the container is closed or not on initialization.
  • data-containment: (string) the DOM element that constrain the container.
  • data-modal: (boolean) define if the container is modal or not.
  • data-buttons: (string) a comma separated list of buttons name (available: fullscreen,dock,close).
  • data-rememberme: (boolean) define if the container state should be stored with cookies or not.
  • data-draggrid: (numbers) it accepts two value separated by comma for the x and y grid (ex: 100,100) for dragging.
  • data-resizegrid: (numbers) it accepts two value separated by comma for the x and y grid (ex: 100,100) for resizing.
  • data-autoresize: (boolean) define if the container should resize when its content changes.

Available Methods

Here is a list of the available methods:

  • open – params: animate, bringToFront: Open a previously closed container.
  • close – params: animate: Close a container.
  • alwaisontop: Make the container always on top of the others.
  • centeronwindow: Center the container on window.
  • collapse: Collapse or restore the container.
  • fullScreen: Expand the container to fill all the screen.
  • iconize: Reduce the container to an icon (or textual) on the “dock” element if specified or on the left of the screen.
  • skin – params: skin (string) the className of the skin Set a new skin for the container.

You can use them as follow:
$([container_ID]).containerize("[method_name]", [params])

Add new methods:

You can easily implement new behaviors or properties for the container using the $.containerize.addMethod(name, fn) method. Adding new methods they will be available for any container and can be applied:

  1. Directly once the container is initialized: add a custom attribute data-[name_of_new_method] to the desired container setting the params as comma separated strings as value of the data.
  2. Invoking the method whenever is needed: passing the name_of_new_method as string parameter of the initialize function: $(“#ID_of_the_container”).containerize(“name_of_new_method”,[params]).

$.containerize.addMethod(name, fn)

Lets make an example on how to use it:

I want to add a method that let me change the content of the container dynamically via AJAX; here is how to write the new method:
The name of this new method is: changecontent


$.containerize.addMethod("changecontent", function(ajaxURL, data){
	$.cMethods.changecontent = {name: "changecontent", author:"pupunzi", type:"plug-in", version:"1.0"};
	var el = this;
	var contentPlaceHolder = el.content;
//	if data is a string convert it to object
        data = data!=null && typeof data == "string" ? eval("("+data+")") : data;
	var request = $.ajax({
		url     : ajaxURL,
		type    : "GET",
		data    : data,
		cache   : false,
		dataType: "html"
	});
	request.done(function (resp) {
		contentPlaceHolder.html(resp);
	});
	request.fail(function (jqXHR, textStatus) {
		contentPlaceHolder.html("error: " + textStatus);
	});

})

$.cMethods.changecontent
This will register the new method descriptor. You can have a list of registered method invoking “$.cMethods” in the browser console.

Inside the method you can refer to the container DOM element with this.

var el = this;
var contentPlaceHolder = el.content;
Each DOM element of the container is available as attribute of the container itself:

  • el.$: the jQuery proxy of the container;
  • el.header: the header;
  • el.containerTitle: the title;
  • el.footer: the footer;
  • el.content: the content;
  • el.buttonBar: the buttonBar;
  • el.toolBar: the toolBar;

ButtonBar

The button bar is the part of the container that contains functional buttons like fullscreen, dock and close.
The buttons display is obtained using the “iconic” font set.

Create and add new buttons

You can create your own button by extending the $.containerize.defaultButtons object before the initialize function as follow:


var newBtns = {
   changecontent: {
     idx:"changecontent",
     label:"&#x2714;",
     className:"",
     action: function(container,btn){$(container).containerize("changecontent","/newContent.php",{id:01, temp:'simple'})}
  }
}
$.extend($.containerize.defaultButtons, newBtns);
  • idx: (string) the unique ID to discriminate the button.
  • label: (string) the label for the button.
  • className: (string) the additional className for the button.
  • action: (function) the function that will be applied on the button click event.

Now you can set changecontent as one of the data-buttons value on the needed containers:
data-buttons = "changecontent, close";

If you need to add buttons after the container initialization you can then use the $.containerize(“addtobuttonbar”, btn, prepend) method:

“addtobuttonbar”: the method used to add the button to the button bar.
btn: (object) the button descriptor.
prepend: (boolean) if the button should placed at the beginning or at the end.

Example:
$.containerize("addtobuttonbar", newBtns.changecontent, true);
This will add the new “changecontent” button as last of the list.

To find out which symbol and which code you can use please see the iconic_fill_demo.html file you can find in css/font/ folder. There you’ll be able to see all the symbols in the iconic set ad get the Hex code for it.
To render the chosen symbol in the HTML code this is the correct syntax:“&#x[char code];”. If, for example, I want to add the symbol that has e044 as Hexadecimal code I have to write: “&#xe044;”.

ToolBar

The toolbar is the part of the container just above the title. This is used to add custom buttons displayed in textual modality to apply special behaviors or actions.

As for the button bar you can add personalized buttons using the $.containerize(“addtotoolbar”, btn, prepend) method:

“addtotoolbar”: the method used to add the button to the tool bar.
btn: (object) the button descriptor.
prepend: (boolean) if the button should placed at the beginning or at the end.

Events:

Each container triggers events for any changes;
here is the list of available events:

  • ready
  • drag
  • dragged
  • resize
  • resized
  • closed
  • opened
  • iconized
  • restored
  • centeredOnWindow
  • collapsed

You can add behaviors on all of that events for each container as follow:
$([the_container_ID]).on(“[one_of_the_above_events]”, function(){ //your code here });
